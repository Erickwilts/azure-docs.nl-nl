## <a name="check-your-azure-synapse-version"></a>Uw Azure Synapse-versie controleren

Controleer de versie die is geïmplementeerd in uw instantie en de nieuwste releasenotes voor beschikbaarheid van functies terwijl nieuwe functies naar alle regio's worden uitgerold. Als u de versie wilt controleren, maakt u verbinding met `SELECT @@VERSION;` uw SQL-groep via SQL Server Management Studio (SSMS) en voert u de huidige versie terug.

Gebruik de geïdentificeerde versie om te bevestigen welke release is toegepast op uw SQL-groep. De datum in de uitvoer identificeert de maand voor de release die is toegepast op uw SQL-groep.

> [!NOTE]
> De productnaam die@VERSION door SELECT @ wordt geretourneerd, wordt gewijzigd van Microsoft Azure SQL Data Warehouse naar Azure Synapse Analytics. We sturen een vervroegde kennisgeving voordat de wijziging wordt aangebracht. Deze wijziging is relevant voor klanten die de productnaam@VERSION ontleeden aan het resultaat van SELECT @ in hun toepassingscode. Om wijzigingen in toepassingscode als gevolg van productrebranding te voorkomen, gebruikt u deze opdrachten om SERVERPROPERTY op te vragen voor de productnaam en -versie van de database: Versienummer XX retourneren. X.XXXXX. X (zonder productnaam) gebruikt deze opdracht:
>
> ```sql
> SELECT SERVERPROPERTY('ProductVersion')
>
> --To return engine edition, use this command that returns 6 for Azure Synapse Analytics (Formerly SQL Data Warehouse):
>
> SELECT SERVERPROPERTY('EngineEdition')
> ```

We hebben nu een extern registratieproces geïntroduceerd via [diagnostische logboeken van Azure Monitor](../../azure-monitor/platform/data-platform.md?toc=/azure/synapse-analytics/sql-data-warehouse/toc.json&bc=/azure/synapse-analytics/sql-data-warehouse/breadcrumb/toc.json#logs) die aanvullende inzichten bieden in de werkbelasting van uw gegevensmagazijn. Met één klik op de knop u nu diagnostische logboeken configureren voor mogelijkheden voor het oplossen van problemen met historische queryprestaties met behulp van [Log Analytics.](../../azure-monitor/log-query/log-query-overview.md?toc=/azure/synapse-analytics/sql-data-warehouse/toc.json&bc=/azure/synapse-analytics/sql-data-warehouse/breadcrumb/toc.json) Azure Monitor diagnostische logboeken ondersteunen aanpasbare bewaarperioden door de logboeken op te slaan in een opslagaccount voor controledoeleinden, de mogelijkheid om logboeken te streamen naar gebeurtenishubs in de buurt van realtime telemetrie-inzichten en de mogelijkheid om logboeken te analyseren met logboekanalyses met logboekquery's. Diagnostische logboeken bestaan uit telemetrieweergaven van uw gegevensmagazijn die gelijkwaardig zijn aan de meest gebruikte prestatieprobleemoplossing voor DMVs voor SQL Analytics in Azure Synapse. |**Geheugenbeheer van columnstore**|Naarmate het aantal rijgroepen voor gecomprimeerde kolomopslag toeneemt, neemt het geheugen toe dat nodig is om de metagegevens van het interne kolomsegment voor die rijgroepen te beheren.  Als gevolg hiervan kunnen queryprestaties en query's die zijn uitgevoerd met een aantal van de Dynamic Management Views (Columnstore Dynamic Management Views) verslechteren.  Verbeteringen hebben in deze release aangebracht om de grootte van de interne metagegevens voor deze gevallen te optimaliseren, wat leidt tot verbeterde ervaring en prestaties voor dergelijke query's.|
