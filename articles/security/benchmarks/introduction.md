# <a name="azure-security-benchmark-introduction"></a>Azure-beveiligingsbenchmark-introductie

U meerdere jaren of zelfs tientallen jaren ervaring hebben met on-premises computing. Je weet hoe je die implementaties beveiligen; maar de wolk is anders. Hoe weet u of uw cloudimplementaties veilig zijn? Wat zijn de verschillen tussen beveiligingspraktijken voor on-premises systemen en cloudimplementaties?

Er is een grote verzameling whitepapers, best practices, referentiearchitecturen, webbegeleiding, open-source tools, commerciële oplossingen, intelligence feeds en meer, die kunnen worden gebruikt om de cloud te beveiligen. Welke optie moet u gebruiken? Wat u doen om een aanvaardbaar beveiligingsniveau in de cloud te krijgen? 

Een van de beste manieren om uw cloudimplementaties te beveiligen, is door u te concentreren op aanbevelingen voor cloudbeveiliging. Benchmarkaanbevelingen voor het beveiligen van elke service beginnen met een fundamenteel inzicht in cyberbeveiligingsrisico's en hoe deze te beheren. U dit inzicht vervolgens gebruiken door benchmarkbeveiligingsaanbevelingen van uw cloudserviceprovider over te nemen om specifieke beveiligingsconfiguratie-instellingen in uw omgeving te selecteren. 

De Azure Security Benchmark bevat een verzameling krachtige beveiligingsaanbevelingen die u gebruiken om de meeste services die u in Azure gebruikt, te beveiligen. U deze aanbevelingen als 'algemeen' of 'organisatorisch' zien omdat ze van toepassing zijn op de meeste Azure-services. De aanbevelingen azure security benchmark worden vervolgens aangepast voor elke Azure-service en deze aangepaste richtlijnen zijn opgenomen in artikelen met serviceaanbevelingen. 

De Azure Security Benchmark-documentatie specificeert beveiligingsbesturingselementen en serviceaanbevelingen.

- **Beveiligingsbesturingselementen:** De aanbevelingen van Azure Security Benchmark worden gecategoriseerd door beveiligingsbesturingselementen. Beveiligingsbesturingselementen vertegenwoordigen standaard vereisten voor leveranciersagnostische beveiliging, zoals netwerkbeveiliging en gegevensbescherming. Elke beveiligingscontrole heeft een reeks beveiligingsaanbevelingen en instructies waarmee u deze aanbevelingen implementeren. 
- **Serviceaanbevelingen:** Indien beschikbaar, bevatten benchmarkaanbevelingen voor Azure-services Azure Security Benchmark-aanbevelingen die specifiek zijn afgestemd op die service. 

De termen 'Controle', 'Benchmark' en 'Basislijn' worden vaak gebruikt in de Azure Security Benchmark-documentatie en het is belangrijk om te begrijpen hoe Azure deze termen gebruikt. 

| Termijn | Beschrijving | Voorbeeld |
|--|--|--|
| Beheer | Een **controle** is een beschrijving op hoog niveau van een functie of activiteit die moet worden aangepakt en niet specifiek is voor een technologie of implementatie. | Gegevensbescherming is een van de beveiligingscontroles. Dit besturingselement bevat specifieke acties die moeten worden aangepakt om ervoor te zorgen dat gegevens worden beschermd. |
| Benchmark | Een **benchmark** bevat beveiligingsaanbevelingen voor een specifieke technologie, zoals Azure. De aanbevelingen worden gecategoriseerd door de controle waartoe ze behoren. | De Azure Security-benchmark bevat de beveiligingsaanbevelingen die specifiek zijn voor het Azure-platform  |
| Basislijn | Een **basislijn** is de beveiligingsvereisten voor een organisatie. De beveiligingsvereisten zijn gebaseerd op benchmarkaanbevelingen. Elke organisatie bepaalt welke benchmarkaanbevelingen ze in hun basislijn willen opnemen. | Het Bedrijf Contoso maakt de beveiligingsbasislijn door te kiezen voor specifieke aanbevelingen in de Azure Security Benchmark. |
