# <a name="azure-functions-and-serverless-platform-security"></a>Beveiliging van Azure Functions en serverloze platforms
## <a name="abstract"></a>Samenvatting
De meeste ondernemingen hebben een aanzienlijke hoeveelheid resources en tijd nodig voor het beheren van servers, waarmee kosten worden toegevoegd. Als ondernemingen minder bronnen kunnen gebruiken om servers te beheren, kunnen ze zich richten op het bouwen van geweldige toepassingen.  

Serverloze computing helpt u gewoon dat, omdat de infra structuur die u nodig hebt om uw apps uit te voeren en te schalen, voor u wordt beheerd. Serverloze computing is de abstractie van servers, infrastructuur en besturingssystemen. Serverloze Computing wordt aangestuurd door de reactie op gebeurtenissen en triggers, die allemaal in bijna realtime worden uitgevoerd, in de Cloud. 

Als volledig beheerde service zijn server beheer en capaciteits planning onzichtbaar voor de ontwikkelaar. Het serverloze Framework helpt u bij het ontwikkelen en implementeren van serverloze toepassingen met behulp van Azure Functions. Het is een CLI (opdracht regel Interface) die structuur en automatisering biedt om u te helpen bij het bouwen van geavanceerde, op gebeurtenissen gebaseerde architecturen die bestaan uit functies en gebeurtenissen. Een Azure-functie is een onafhankelijke implementatie-eenheid, zoals een micro service. Het is alleen code, geïmplementeerd in de Cloud, die het vaakst wordt geschreven om één taak uit te voeren.

Ondanks de voor delen heeft serverloze beveiliging zijn eigen risico factoren om met te kunnen omgaan. De serverloze benadering introduceert geen nieuwe beveiligings problemen, maar vereist een aanpak van bestaande beveiligings problemen. Dit technisch document richt zich op deze beveiligings aangelegenheden: 
* Voor delen van een serverloos platform
* Beveiligings problemen in serverloze computing
* Kritieke beveiligings problemen en-oplossingen in de context van Azure
* Het micro soft serverloze platform beveiligen
