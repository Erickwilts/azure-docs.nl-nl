Hier vindt u de beperkingen voor gebruik en andere servicelimieten voor de Azure Active Directory-service (Azure AD).

| Categorie | Limiet |
| --- | --- |
| Mappen | Eén gebruiker kan als lid of gast deel uitmaken van maximaal 500 Azure AD-directory's.<br/>Een enkele gebruiker kan maximaal 200 mappen maken. |
| Domeinen | U kunt niet meer dan 900 beheerde domeinnamen toevoegen. Als u al uw domeinen wilt instellen voor federatie met on-premises Active Directory, kunt u niet meer dan 450 domeinnamen toevoegen in elke directory. |
|Resources |<ul><li>Gebruikers van de gratis editie van Azure Active Directory kunnen standaard maximaal 50.000 Azure AD-resources in één map maken. Als u ten minste één geverifieerd domein hebt, wordt het standaard A2-servicequotum voor uw organisatie uitgebreid tot 300.000 Azure AD-resources. Deze servicelimiet staat los van de prijscategorielimiet van 500.000 resources op de prijspagina van Azure AD. Als u verder wilt gaan dan het standaardquotum, moet u contact opnemen met Microsoft Support.</li><li>Een niet-beheerdersgebruiker kan niet meer dan 250 Azure AD-resources maken. Zowel actieve resources als verwijderde resources die beschikbaar zijn om het aantal bronnen voor dit quotum te herstellen. Alleen verwijderde Azure AD-resources die minder dan 30 dagen geleden zijn verwijderd, kunnen worden hersteld. Verwijderde Azure AD-resources die niet langer beschikbaar zijn om het aantal voor dit quotum te herstellen met een waarde van één kwartaal gedurende 30 dagen. Als u ontwikkelaars hebt die dit quotum waarschijnlijk herhaaldelijk zullen overschrijden in de loop van hun reguliere taken, u [een aangepaste rol maken en toewijzen](../articles/active-directory/users-groups-roles/roles-quickstart-app-registration-limits.md) met toestemming om een onbeperkt aantal app-registraties te maken.</li></ul> |
| Schema-uitbreidingen |<ul><li>Uitbreidingen van het type 'tekenreeks' mogen uit maximaal 256 tekens bestaan. </li><li>Uitbreidingen van het type 'binair' mogen uit maximaal 256 bytes bestaan.</li><li>Slechts 100 extensiewaarden, voor *alle* typen en *alle* toepassingen, kunnen worden geschreven naar één Azure AD-bron.</li><li>Alleen de entiteiten User, Group, TenantDetail, Device, Application en ServicePrincipal mogen worden uitgebreid met kenmerken met één waarde van het type 'tekenreeks' of 'binair'.</li><li>Schema-uitbreidingen zijn alleen beschikbaar in de preview van Graph API-versie 1.21. Er moet schrijftoegang worden verleend aan de toepassing om een uitbreiding te kunnen registreren.</li></ul> |
| Toepassingen |Maximaal 100 gebruikers mogen eigenaar zijn van één toepassing. |
|Toepassingsmanifest |In het Application Manifest kunnen maximaal 1200 inzendingen worden toegevoegd. |
| Groepen |<ul><li>Een gebruiker kan maximaal 250 groepen maken in een Azure AD-organisatie.</li><li>Een Azure AD-organisatie kan maximaal 5000 dynamische groepen hebben.<li>Maximaal 100 gebruikers mogen eigenaar zijn van één groep.</li><li>Elk aantal Azure AD-resources kan lid zijn van één groep.</li><li>Een gebruiker kan lid zijn van een willekeurig aantal groepen.</li><li>Het aantal leden in een groep dat u met behulp van Azure AD Connect vanaf uw on-premises Active Directory naar Azure Active Directory kunt synchroniseren, is beperkt tot 50.000.</li><li>Geneste groepen in Azure AD worden niet binnen alle scenario's ondersteund</li></ul><br/> Op dit moment zijn de volgende de ondersteunde scenario's met geneste groepen.<ul><li> Een groep kan worden toegevoegd als lid van een andere groep en u groepsnesten bereiken.</li><li> Groepslidmaatschapclaims (wanneer een app is geconfigureerd om groepslidmaatschapsclaims in het token te ontvangen, zijn geneste groepen waarvan de aangemelde gebruiker lid is)</li><li>Voorwaardelijke toegang (bij het scoping van een beleid voor voorwaardelijke toegang tot een groep)</li><li>Toegang tot het opnieuw instellen van zelfbediening wachtwoorden beperken</li><li>Beperken welke gebruikers Azure AD Join en apparaatregistratie kunnen doen</li></ul><br/>De volgende scenario's worden niet ondersteund geneste groepen:<ul><li> Toewijzing van een app-rol (het toewijzen van groepen aan een app wordt ondersteund, maar groepen die zijn genest binnen de direct toegewezen groep hebben geen toegang), zowel voor toegang als voor inrichting</li><li>Licenties op basis van groepen (automatisch een licentie toewijzen aan alle leden van een groep)</li><li>Office 365-groepen.</li></ul> |
| Toepassingsproxy | <ul><li>Maximaal 500 transacties per seconde per App Proxy-toepassing</li><li>Maximaal 750 transacties per seconde voor de Azure AD-organisatie</li></ul><br/>Een transactie wordt gedefinieerd als één http-aanvraag en -antwoord voor een unieke bron. Wanneer klanten worden beperkt, ontvangen ze een antwoord van 429 (te veel aanvragen). |
| Toegangsvenster |<ul><li>Er is geen limiet voor het aantal toepassingen dat per gebruiker kan worden weergegeven in het toegangsvenster. Dit geldt voor gebruikers met licenties voor Azure AD Premium of de Enterprise Mobility Suite.</li><li>Er worden maximaal 10 app-tegels weergegeven in het toegangsvenster voor iedere gebruiker. Deze limiet geldt voor gebruikers die licenties hebben toegewezen voor het Azure AD Free-licentieplan. Voorbeelden van app-tegels zijn Box, Salesforce en Dropbox. Deze limiet geldt niet voor beheerdersaccounts.</li></ul> |
| Rapporten | Er kunnen maximaal 1000 rijen worden bekeken of gedownload in elk rapport. Aanvullende gegevens worden afgekapt. |
| Beheereenheden | Een Azure AD-bron kan lid zijn van niet meer dan 30 administratieve eenheden. |
| Beheerdersrollen en -machtigingen | <ul><li>Een groep kan niet worden toegevoegd als [eigenaar.](https://docs.microsoft.com/azure/active-directory/fundamentals/users-default-permissions?context=azure/active-directory/users-groups-roles/context/ugr-context#object-ownership)</li><li>Een groep kan niet aan een [rol](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles)worden toegewezen .</li><li>De mogelijkheid van gebruikers om de adreslijstgegevens van andere gebruikers te lezen, kan niet worden beperkt buiten de overschakeling in de azure AD-organisatie om de toegang van alle niet-beheerdersgebruikers tot alle directorygegevens uit te schakelen (niet aanbevolen). Meer informatie over standaardmachtigingen [hier](https://docs.microsoft.com/azure/active-directory/fundamentals/users-default-permissions?context=azure/active-directory/users-groups-roles/context/ugr-context#to-restrict-the-default-permissions-for-member-users).</li><li>Het kan tot 15 minuten duren of u afmelden/aanmelden voordat toevoegingen en intrekkingen van het lidmaatschap van beheerdersrollen van kracht worden.</li></ul> |
