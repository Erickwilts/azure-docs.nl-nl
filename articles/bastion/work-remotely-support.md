# <a name="working-remotely-using-azure-bastion"></a>Op afstand werken met Azure Bastion

Azure Bastion speelt een cruciale rol bij het ondersteunen van werkscenario's op afstand door gebruikers met een internetverbinding toegang te geven tot virtuele Azure-machines. Het stelt IT-beheerders in staat om hun toepassingen die op Azure worden uitgevoerd op elk gewenst moment en overal ter wereld te beheren.

>[!NOTE]
>In dit artikel wordt beschreven hoe u Azure Bastion, Azure, Microsoft-netwerk en het Azure-partnerecosysteem gebruiken om op afstand te werken en netwerkproblemen te beperken waarmee u wordt geconfronteerd vanwege de COVID-19-crisis.
>

## <a name="securely-access-virtual-machines"></a>Veilig toegang tot virtuele machines

Azure Bastion biedt met name veilige en naadloze RDP/SSH-connectiviteit met virtuele machines binnen het virtuele Azure-netwerk, rechtstreeks in de Azure-portal, zonder het gebruik van een openbaar IP-adres. Ga voor meer informatie over de Azure Bastion-architectuur en de belangrijkste functies naar [Wat is Azure Bastion](bastion-overview.md).

Azure Bastion wordt geïmplementeerd per virtueel netwerk, wat betekent dat bedrijven één Azure Bastion kunnen configureren en beheren om externe gebruikerstoegang tot virtuele machines binnen een virtueel Azure-netwerk snel te ondersteunen. Raadpleeg voor richtlijnen voor het maken en beheren van Azure Bastion [een bastionhost maken.](bastion-create-host-portal.md)
