# <a name="overview-of-azure-security-baselines"></a>Overzicht van Azure Security Baselines

Azure Security Baselines helpen u de beveiliging van onze producten te verbeteren door verbeterde hulpprogramma's, tracking en beveiligingsfuncties en bieden u een consistente ervaring bij het beveiligen van uw omgeving.

Azure Service Security Baselines richten zich op cloudgerichte besturingsgebieden. Deze besturingselementen zijn in overeenstemming met bekende beveiligingsbenchmarks, zoals beschreven door het Center for Internet Security (CIS). Onze basislijnen bieden richtlijnen voor de controlegebieden die worden vermeld in de [Azure Security Benchmark.](overview.md)

Elke aanbeveling bevat de volgende informatie:
- **Azure ID:** de Azure Security Benchmark ID die overeenkomt met de aanbeveling.
- **Aanbeveling:** Direct na de Azure ID geeft de aanbeveling een beschrijving op hoog niveau van het besturingselement.
- **Richtsnoeren**: De motivering van de aanbeveling en de richtsnoeren voor de wijze waarop deze moeten worden uitgevoerd. Als de aanbeveling wordt ondersteund door Azure Security Center, wordt die informatie ook weergegeven.
- **Verantwoordelijkheid**: Wie is verantwoordelijk voor de uitvoering van de controle. Mogelijke scenario's zijn verantwoordelijkheid van de klant, de verantwoordelijkheid van Microsoft of gedeelde verantwoordelijkheid.
- **Azure Security Center-bewaking:** of het besturingselement wordt gecontroleerd door Azure Security Center, met koppeling naar referentie.

Alle aanbevelingen, inclusief aanbevelingen die niet van toepassing zijn op deze specifieke service, worden opgenomen in de basislijn om u een volledig beeld te geven van hoe de Azure Security Benchmark zich verhoudt tot elke service. Er kunnen af en toe besturingselementen zijn die om verschillende redenen niet van toepassing zijn, bijvoorbeeld, IaaS/compute-centric besturingselementen (zoals besturingselementen die specifiek zijn voor osconfiguratiebeheer) mogelijk niet van toepassing zijn op PaaS-services.
